Semantic Analyser

Explore the third phase of a compiler — the Semantic Analyser. Understand how a compiler validates the meaning of source code beyond syntax, including type checking, scope resolution, undeclared variable detection, and type compatibility enforcement using a Symbol Table.

🔍 Type Checking

Verifies that operators and operands are type-compatible. Detects errors like adding an integer to a string or assigning a float to a char without proper casting.

📋 Symbol Table

Maintains a data structure that stores information about every identifier — its name, type, scope, and memory location — used throughout the compilation process.

🚨 Error Detection

Catches semantic errors invisible to the parser: undeclared variables, duplicate declarations, type mismatches, scope violations, and incompatible assignments.

🔄 Type Coercion

Handles implicit type conversions (widening) such as automatically promoting an int to float in mixed-type expressions, following the type hierarchy rules.

🎯 Scope Resolution

Ensures variables are used within their valid scope. Detects when a variable declared inside a block is accessed outside, or when names collide across scopes.

🌳 Annotated AST

Produces an Annotated Abstract Syntax Tree where each node carries type information, enabling the next phases (intermediate code generation) to proceed correctly.

Theory

Understanding the fundamentals of Semantic Analysis

What is a Semantic Analyser?

A Semantic Analyser is the third phase of a compiler that checks the meaning of the source code. While the Lexical Analyser validates tokens and the Syntax Analyser validates structure, the Semantic Analyser ensures that the program makes logical sense. It uses the parse tree produced by the parser and enriches it with type information using a Symbol Table.


Key Functions

  • Type Checking — ensures operands and operators are compatible
  • Declaration Verification — ensures every variable is declared before use
  • Duplicate Detection — prevents re-declaring a variable in the same scope
  • Scope Resolution — resolves variable references to the correct declaration
  • Type Coercion — inserts implicit type conversions where necessary
  • Symbol Table Management — builds and maintains a table of all identifiers
  • Annotated AST Generation — attaches type info to every tree node

How It Works

The semantic analyser traverses the Abstract Syntax Tree (AST) produced by the parser. At each node, it performs checks:

  • When a declaration is encountered (e.g., int x;), the variable is added to the Symbol Table
  • When a variable is used (e.g., x = 5;), the analyser checks the Symbol Table to verify it was declared
  • When an expression is evaluated (e.g., x + y), the types of both operands are checked for compatibility
  • When an assignment occurs (e.g., x = y + 3.14;), the type of the right-hand side must be compatible with the left-hand side

Example

  int a = 10;
  float b = 3.14;
  char c = "hello";

  a = a + b;      // ✅ OK: int + float → float (type coercion)
  b = a * 2;      // ✅ OK: int * int → int, assigned to float
  a = c + 1;      // ❌ ERROR: char (string) + int → type mismatch
  d = a + b;      // ❌ ERROR: variable 'd' not declared
  int a = 20;     // ❌ ERROR: duplicate declaration of 'a'

Semantic Checks

Types of semantic errors detected by the analyser

Check Type Description Example Error
Undeclared Variable Variable used without prior declaration x = 10; (x was never declared)
Duplicate Declaration Variable declared more than once in same scope int a; int a;
Type Mismatch (Assignment) Assigning an incompatible type to a variable int x = "hello";
Type Mismatch (Expression) Using incompatible types in an operation char a = "hi"; a + 5;
Type Coercion (Implicit) Automatic widening conversion applied float x = 10; (int → float)
Narrowing Warning Assigning a wider type to a narrower one int x = 3.14; (float → int, data loss)

Symbol Table

The core data structure used in semantic analysis

What is a Symbol Table?

A Symbol Table is a data structure maintained by the compiler that stores information about each identifier (variable, function, etc.) encountered in the source code. It is built during the semantic analysis phase and used throughout the remaining compilation process.


Symbol Table Entry Fields

Field Description Example
Name The identifier name count
Type Data type of the variable int, float, char
Scope Where the variable is visible global, local
Line Declared Line number where first declared 3
Value Assigned value (if initialization present) 10, 3.14, "hello"

Type System

Type hierarchy and compatibility rules

Type Hierarchy (Widening Order)

   char  →  int  →  float  →  double
   (narrowest)                (widest)

   Widening (implicit):  int → float   ✅ Safe
   Narrowing (warning):  float → int   ⚠️ Data loss possible

Type Compatibility Matrix

Operation int ⊕ int int ⊕ float float ⊕ float char ⊕ int char ⊕ char
+ - * / ✅ int ✅ float (coerce) ✅ float ❌ error ❌ error
= (assign) ✅ int ⚠️ narrowing ✅ float ❌ error ✅ char
